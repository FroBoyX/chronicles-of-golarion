# Source Spines

This folder holds shared factual spines for narrative output when an existing chronology/day/event file is not enough.

A source spine is the controlled source layer for a day or event. The three sibling output types derive from it:

1. Ship's Log
2. Narrative Write-Up
3. Narrative Chat Scene

Do not draft one sibling output from another sibling output. Draft each from the spine.

Use `design/narrative-output-method.md` for the required source-spine fields and workflow.

Recommended filename pattern:

```text
<date-or-event-slug>.md
```

Examples:

```text
erastus-30-mistress-mirei.md
mans-promise-boarding.md
bonewrack-hot-box.md
```

Only create a dedicated source spine when a target day/event needs aligned outputs or when the existing chronology file does not already contain the required source-index, quote-ledger, character-knowledge, locked-fact, open-check, and sibling-alignment sections.
