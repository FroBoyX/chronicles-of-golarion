# Chronicle Initiative Roadmap

## Product Goal

Make **Chronicles of Golarion** runnable from the repository alone.

The immediate goal is not to finish documenting the whole campaign. The immediate goal is to support the relaunch at the Bonewrack fork with enough confidence that the only uncertainty at the table is what the players choose.

## Version 0.1 — Archive Foundation

**Goal:** The repository exists and has rules for canon, evidence, and process.

### Complete

- Repository initialized.
- README charter established.
- Canon hierarchy established.
- Contribution rules established.
- Chronicle Method established.
- Current campaign state seeded.
- Open threads seeded.
- Mirei dossier seeded.
- Disease rules recorded.
- Initial NPC dossiers seeded for Mr. Plugg and Ambrose Kroop.

### Remaining

- Repository index.
- Document templates.
- Entity ID convention.

## Version 0.2 — Island Relaunch

**Goal:** A GM can resume play at the Bonewrack fork.

### Success Criteria

- The current split-party state is complete.
- Bonewrack Isle is reconstructed as actually played.
- The Man's Promise is documented as a current political and physical space.
- Immediate NPCs have usable dossiers.
- The decision space after the island is documented without forcing an outcome.
- The campaign can support multiple outcomes: PCs take the ship, Plugg retains control, the crew fractures, the ship is abandoned, or some other state emerges from play.

## Version 1.0 — Relaunch Complete

**Goal:** The campaign can resume from the repository alone.

A GM should be able to open the repository and answer:

- Where is every important character?
- What does each important character want?
- What does each important character know?
- What is the state of the ship?
- What is the state of the crew?
- What decisions are available?
- What consequences follow from each broad outcome?

## Core Rule

The archive must not decide the players' future.

It documents the state from which that future emerges.
