# PF2e Conversion Index

## Document Status

- Type: Conversion tracking document
- Maturity: Working index
- Scope: PF1 to PF2e conversion targets, candidates, approvals, rejected paths, and open conversion risks

## Purpose

This file tracks possible Pathfinder 2e relaunch mechanics without overwriting Pathfinder 1e historical evidence.

PF2e conversion status does not change campaign canon. It only tracks possible or approved mechanics for future play.

## Status Values

- **Not started** — identified target, no audit yet.
- **Needs source review** — PF1 evidence or character truth not stable enough.
- **Draft audit** — conversion brief started.
- **Candidate proposed** — one or more PF2e paths proposed.
- **Player review needed** — PC conversion requires player review.
- **GM review needed** — awaiting GM mechanical/canon approval.
- **Approved relaunch mechanics** — adopted for future PF2e play.
- **Rejected** — conversion path rejected.
- **Superseded** — replaced by later conversion candidate.

## Notes

- Do not approve PC conversion without player review.
- Do not convert PF1 numbers directly.
- Do not let PF2e conversion imply that historical PF1 outcomes should change.
- Conversion can be adopted, deferred, or rejected after pilot testing.
- Use PF2e-native rules first while the GM is still building PF2e fluency, except where GM combat-design intent must be preserved and carefully fitted into PF2e.
- Tactical facing is a draft homebrew subsystem candidate. It should be tested in a contained PF2e pilot before campaign-wide adoption.
- Active Defense is a draft homebrew subsystem candidate. It should be tested only after baseline PF2e and the Facing Variant are stable enough to evaluate cleanly.
- Affliction templates are draft conversion templates. They should be used to build a few sample diseases and poisons before any campaign-wide adoption.
- Skill Unlocks are preserved as conversion evidence and table style. Current recommendation is to use PF2e-native skill proficiency and skill feats rather than porting PF1 unlock progressions directly.
- Stamina Pool / Combat Tricks are preserved as conversion evidence and table style. Current recommendation is to defer them until after baseline PF2e, Facing Variant, and Active Defense are tested.
- Wound Thresholds are preserved as conversion evidence and table style. Current recommendation is to defer them and use PF2e wounded/dying rules, visible injury narration, and encounter-specific bloodied behavior first.
