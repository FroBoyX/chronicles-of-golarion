# PF2e Skill Unlocks Conversion Policy

## Document Status

- Type: PF2e campaign subsystem conversion note
- Maturity: Draft / conversion policy
- PF2e status: Use PF2e-native skill proficiency and skill feats; do not port PF1 unlocks directly
- Spoiler scope: Player-safe rules draft
- Created: 2026-07-07

## Purpose

This document preserves the table's Pathfinder 1e Skill Unlocks usage as conversion evidence and maps its intent to Pathfinder 2e.

The original table rule came from Pathfinder Unchained and gave characters new skill uses at 5, 10, 15, and 20 ranks in a skill. This file does not port those unlocks directly. It records how their design intent should be handled during PF2e relaunch conversion.

This file does not rewrite PF1 historical evidence, prior rolls, prior outcomes, or campaign canon.

## PF1 Historical / Table Basis

The table had Skill Unlocks active.

The uploaded rule text describes Skill Unlocks as Pathfinder Unchained rules that give new abilities and skill uses at 5, 10, 15, and 20 ranks in a skill. Access could come through the Signature Skill feat, rogue's edge, or a universal campaign option. The source text also warns that universal access increases the power and flexibility of skills, especially for already skilled classes.

The uploaded list includes unlock progressions for skills such as:

- Acrobatics;
- Appraise;

## PF2e Source Basis Checked

Rules basis checked against Archives of Nethys PF2e pages on 2026-07-07:

- Skills — Player Core pg. 225; skills represent training and experience, are keyed to attributes, and include actions that may be untrained or require training.
- Skill Actions — some skill actions require trained proficiency or higher, and the GM can decide when a task requires a particular proficiency rank.
- Skill Increases — characters improve skills by skill increases, becoming trained, expert, master, or legendary, with master normally requiring level 7+ and legendary normally requiring level 15+.
- Skill Feats — skill feats are general feats with the skill trait that give new ways to use skills or make characters better at particular skill uses.
- Skill DCs — simple DCs include trained, expert, master, and legendary benchmarks.
- Archetypes — some archetype feats have the skill trait and can be taken in place of skill feats.

## Design Position

PF2e already contains the main design solution Skill Unlocks were trying to provide.

Do not port PF1 Skill Unlocks directly.

Instead, preserve the intent through:

- PF2e proficiency rank gating: trained, expert, master, legendary;
- PF2e skill feats;
- class features that grant skill increases or skill feats;
- archetype skill feats when appropriate;
- GM permission for high-proficiency tasks that require expert, master, or legendary training;
- conversion notes that identify which old PF1 unlocks were part of a character's table identity.

## Why Direct Port Is Not Recommended

Directly porting PF1 Skill Unlocks would duplicate PF2e's native skill architecture.

Risks:

- too many extra skill abilities layered on top of PF2e skill feats;
- excessive power boost for high-skill characters;
- overlap with existing skill feats, class feats, and archetypes;
- confusion between PF1 ranks and PF2e proficiency ranks;
- unnecessary rebuild burden during PC conversion;
- Foundry implementation friction if custom unlocks must be tracked manually.

## Conversion Rule

For PC and NPC conversion:

1. Identify any PF1 Skill Unlocks the character actually used or that mattered to table identity.
2. Do not grant the PF1 unlock automatically.
3. Map the old unlock's intent to one of:
   - a PF2e skill feat;
   - a higher proficiency rank;
   - a class feature;
   - an archetype feat;
   - a custom campaign feat only if no PF2e-native option preserves the concept.
4. Record unresolved gaps in the character's PF2e conversion brief.
5. Do not create a universal free Skill Unlock progression unless a later pilot shows PF2e skills feel underpowered for the campaign.

## Campaign Use Notes

Skill Unlocks supported a table style where long-term investment in a skill created new permissions and distinctive competence.

That identity should survive conversion.

For *Skull & Shackles*, this matters especially for:

- ship roles and labor competence;
- climbing, swimming, balancing, and shipboard movement;
- social deception, intimidation, command, and diplomacy;
- medicine, poison, disease, and survival pressure;
- stealth and ambushes;
- item identification, treasure evaluation, and scavenging;
- Lore-based sailor, pirate, cook, surgeon, quartermaster, gunner, and navigator identities.

## Recommended Pilot

Do not add a universal PF2e Skill Unlock subsystem during the first conversion pilot.

Instead:

1. Convert one PC using PF2e-native skill proficiency and skill feats.
2. Identify any old Skill Unlock identity pressure.
3. Confirm whether PF2e skill feats already cover it.
4. If not, write a targeted custom campaign feat or ruling for that specific gap.

Likely pilot questions:

- Does Sho need special Acrobatics / Athletics / Sense Motive-style competence beyond PF2e feats?
- Does Rak need special Crafting / Medicine / poison / cooking / alchemy / Sleight of Hand identity support?
- Does Mirei need Performance / Diplomacy / sea-lore skill expression beyond Bard features?
- Does Harvus need Athletics / Intimidation / Survival / flight-climb movement identity support?
